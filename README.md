# 二手裝備門市系統 — 開發文件包

本文件包是給 **Claude Code** 進行實作的規格來源。實作前請完整讀過 `CLAUDE.md` 與本資料夾全部文件，並嚴格遵守當中的約束（TDD、專案結構、PII 保護、發票開關、多分店就緒等）。

## 系統定位

一間（未來可能擴張為多間）販售與收購二手裝備的門市，業務包含：

- **收購（買斷）** 二手裝備：店家付現金買下客人的裝備，成為店家庫存。
- **寄售** 帳篷：貨權屬於寄售人，賣出後依抽成比例拆帳（預設店家抽 50%）。
- **轉售**：把二手裝備（買斷／寄售）賣給客人。
- **販售全新商品與飲料**：向供應商進貨的一般商品。
- 全程**只收現金**；依法開立**電子發票**（透過自建/雲端 Turnkey 上傳），但提供系統開關控制是否開立。

> ⚠️ **叫號機不在開發範圍內**（店家另行採購硬體）。收購流程本身照常運作，僅不含叫號整合，但保留乾淨邊界供未來整合。

## 技術選型摘要

- **後端**：Python 3.12+ / FastAPI / SQLAlchemy 2.0 / Pydantic v2 / Alembic
- **資料庫**：PostgreSQL（容器化，自動備份；未來可換雲端託管）
- **前端**：Next.js（App Router）/ TypeScript
- **硬體代理**：獨立 Python 服務，於 POS 機器上以 localhost 提供列印/錢櫃控制（ESC/POS）
- **部署**：Docker Compose 於店內伺服器；Turnkey 另一機器或雲端 VM
- **開發方法**：TDD（test-first，紅→綠→重構）

## 重要前提（請店家/會計師確認，非工程範圍）

1. **電子發票稅務**：二手收購（向個人取得無進項憑證）、二手轉售、寄售拆帳的稅務與發票開立方式，請會計師確認；系統把稅率與處理方式做成可設定，不寫死。
2. **發票開關**：系統允許關閉發票開立（草創/測試期），但**所有銷售一律完整記錄**，開關只控制是否產生並上傳電子發票。以營業人身分長期關閉是否合規，請會計師確認。
3. **個資**：收購/寄售強制蒐集姓名、身分證字號屬高度敏感個資，系統會加密儲存、限存取、留稽核；蒐集時的告知義務（個資法）由店家流程負責。
