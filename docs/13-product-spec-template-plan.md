# 13 — 商品分類與規格範本規劃（待 Phase 3 後實作）

> 狀態：規劃存檔，尚未實作。預定在 Phase 3（POS／發票／硬體）完成後進行。
> 實作時須遵守 docs/12（git 流程）、CLAUDE.md（所有準則）、TDD，且因會修改既有
> T5(inventory)，必須含完整回歸測試，確保不破壞 T7(收購)／現金等既有功能。不得擅自更動
> 已定案規則。

## 一、目標
為每件露營商品提供結構化、可篩選的規格資訊，支援店員建檔與日後客人瀏覽網站（查看當前庫存、
依分類篩選）。採「範本驅動」：每個商品類別定義自己的規格欄位與必填／可選規則。

## 二、已確認的設計決策
1. 範本驅動：每類一份規格範本，定義該類有哪些規格欄位、型別、必填／可選、是否可篩選。
2. 分類兩層：大分類 → 小分類，由店長（MANAGER）後台預先定義維護；店員（CLERK）能否新增／
   管理分類與範本，由 settings 開關 `allow_clerk_manage_categories` 控制（MANAGER 恆可）。
3. 品項狀況取代 grade 字母制：移除 S／A／B／C／D／E 字母制，改為共用必填欄位「品項狀況」，
   預設選項：全新未拆封／全新已拆封／九成新／八成新／七成新／五成新，且店長可自定義增減。
4. 散裝觸發改變：原本 grade=E 觸發 bulk_lot（散裝批），改為「建檔時明確選擇『散裝』」來觸發；
   字母制移除後，散裝不再依賴 grade。
5. 「熱門」自動化：原 grade 的 S=熱門改為由銷售資料自動統計（當月／當季熱賣類別或品項），
   放 Phase 6 報表，不再由店員手動標記。
6. 必填依賣家類型：向個人買斷／寄售 → 強制姓名 + 身分證字號（沿用現有 PII 加密／blind index）；
   向廠商進貨（新品）→ 用廠商資料、免身分證。
7. 篩選粒度從簡（參考 MOMO 等主流網站）：客人篩選以「分類 + 價格 + 品項狀況 + 品牌」為主，
   再加極少數關鍵標籤（帳篷：適用人數；桌／爐：是否相容 IGT）。其餘規格（耐水壓、丹數、流明、
   續航等）僅作商品詳情展示，不做篩選條件，且對二手品幾乎全為可選。

## 三、共用欄位（所有商品，不分類別）
必填：品牌、品名（autocomplete）、大分類、小分類、品項狀況、售價。
買斷時另填：收購價。
可選：照片、顏色、重量、商品描述／備註。
賣家：個人 → 姓名+身分證（必填）；廠商 → 廠商資料（免身分證）。

## 四、分類清單（起手，店長可增刪）
- 帳篷類（小：寢室帳／客廳帳／速開帳／天幕／車頂帳）
- 桌子（小：IGT 桌／蛋捲桌／折疊桌／小邊桌）
- 椅子（小：折疊椅／月亮椅／躺椅／充氣椅／兒童椅）
- 營燈／照明（小：露營燈／頭燈／手電筒／燈架）
- 睡袋
- 睡墊／充氣床（小：充氣床／自動充氣墊／泡棉墊／行軍床）
- 爐具（小：卡式爐／IGT 爐／蜘蛛爐／焚火台）
- 保冷保溫（小：冰桶／保冷袋／行動冰箱）
- 收納／裝備袋
- 寢具／枕頭
- 配件／其他

## 五、各類規格範本（欄位；除共用必填外一律可選；標【篩】＝客人篩選條件）
- 帳篷：適用人數【篩】、帳型、是否含內帳、展開尺寸、收納尺寸、重量、附屬配件（可多選）；
  詳情選填：外帳耐水壓(mm)、內帳／底布耐水壓(mm)、營柱材質、布料丹數(D)。
- 桌子（含 IGT）：桌型、材質；IGT 專屬：單位數(1／2／3／4／4.5／5)、是否含桌板、桌板數量、
  桌板品牌、是否滑軌通用、承重(kg)、高度可調範圍、展開／收納尺寸、重量；是否相容 IGT【篩】。
- 椅子：椅型、承重(kg)、材質、展開／收納尺寸、重量。
- 營燈／照明：光源(LED／瓦斯)、流明(lm)、續航(小時)、供電(充電／電池／瓦斯)、色溫／可調光、
  防水等級、重量。
- 睡袋：舒適溫度(°C)、填充材(羽絨／化纖)、形狀(信封／木乃伊)、展開／收納尺寸、重量。
- 睡墊／充氣床：類型(充氣／自動充氣／泡棉／行軍床)、尺寸、厚度、R 值、重量。
- 爐具：類型、火力(W／kcal)、燃料(瓦斯／卡式罐／汽油)、是否可嵌入 IGT【篩】、重量。
- 保冷保溫：容量(L)、材質、尺寸、是否折疊。
- 收納／裝備袋：容量(L)、材質、尺寸、是否折疊。
- 天幕：尺寸、形狀、布料／耐水壓、附配件。
- 其餘類別比照，店長後台定義。

## 六、資料模型方向（實作時細化）
- category：兩層（parent／child），帶 store_id，店長維護。
- spec_template：掛在 category 上，定義欄位（名稱／型別：數字｜文字｜單選｜多選／必填／
  可篩選／選項）。
- product_model／serialized_item：存實際規格值，採可查詢的結構化存法（非純自由 JSON），
  以支援篩選。
- 移除 grade 字母制 → 新增「品項狀況」共用必填欄位；散裝改由建檔時選「散裝」觸發 bulk_lot。
- settings 新增 allow_clerk_manage_categories。
- 熱賣統計放 Phase 6 報表。

## 七、建檔手順（店員）
1. 選／建賣家（個人→姓名+身分證；廠商→免身分證）。
2. 選大分類 → 小分類（系統據此套該類規格範本）。
3. 填品名（autocomplete；選既有型號→自動帶入品牌／分類／規格／價格歷史，可微調）。
4. 動態規格表單（依類別長出欄位；共用必填標星號，規格欄位多為可選）+ 選品項狀況。
5. 填收購價 → 定價輔助算建議售價（毛利率，沿用 core/money）→ 店員可手動調。
6. 選填拍照 → 確認入庫 → 產 item_code、印標籤；買斷付現則同步現金出帳（沿用 T7）。
